# 私人聊天功能与端到端加密

## 概述

本文档概述了带有端到端加密的私人聊天功能设计，确保用户之间的消息是安全的，只能被预期的接收者阅读。

## 密钥生成和交换

### 1. 密钥对生成
- 每个用户生成唯一的非对称密钥对（公钥和私钥）
- 使用Signal协议的实现生成密钥
- 私钥安全存储在设备的Android KeyStore中
- 公钥与服务器共享，以便分发给其他用户

### 2. 预密钥交换
- 用户生成一组预密钥（一次性使用的公钥）
- 预密钥上传到服务器
- 当启动新聊天时，发起用户检索接收者的预密钥

### 3. 会话建立
- **X3DH密钥协商**: 使用Signal协议的扩展三重Diffie-Hellman (X3DH)进行安全会话建立
- **会话密钥**: 派生共享会话密钥用于加密消息
- **棘轮机制**: 实现Double Ratchet算法以提供前向保密性

## 消息加密过程

### 1. 消息准备
- 用户在聊天界面中撰写消息
- 应用检索接收者的公钥和当前会话状态

### 2. 加密
- **消息密钥派生**: 使用当前会话状态派生消息密钥
- **AES-256加密**: 使用派生密钥加密消息内容
- **消息认证**: 生成HMAC以确保消息完整性
- **棘轮更新**: 使用Double Ratchet算法更新会话状态

### 3. 消息传输
- 加密消息发送到服务器
- 服务器将消息路由到接收者
- 服务器存储加密消息而不访问其内容

## 消息解密过程

### 1. 消息接收
- 接收者的设备接收加密消息
- 应用识别发送者并检索相应的会话状态

### 2. 解密
- **会话状态检索**: 加载发送者的会话状态
- **消息密钥派生**: 使用会话状态派生相同的消息密钥
- **AES-256解密**: 解密消息内容
- **完整性验证**: 验证消息HMAC
- **棘轮更新**: 使用Double Ratchet算法更新会话状态

### 3. 消息显示
- 解密的消息显示在聊天界面中
- 消息状态更新为"已送达"和"已读"

## 安全考虑

### 1. 前向保密性
- **Double Ratchet算法**: 确保即使长期密钥被泄露，过去的消息仍然安全
- **临时密钥**: 使用使用后丢弃的临时密钥

### 2. 认证
- **数字签名**: 使用Ed25519签名验证消息真实性
- **密钥验证**: 用户可以通过QR码或密钥指纹验证彼此的公钥

### 3. 隐私
- **服务器不知情**: 服务器无法读取消息内容
- **元数据保护**: 最小化存储在服务器上的元数据
- **完美前向保密性**: 泄露的密钥不会影响过去的通信

### 4. 密钥管理
- **安全存储**: 私钥存储在Android KeyStore中
- **密钥轮换**: 定期轮换密钥以维持安全性
- **密钥备份**: 可选的密钥备份用于消息恢复

## 实现细节

### 1. 前端实现
- **Signal协议库**: 集成libsignal-protocol-java
- **密钥存储**: 使用Android KeyStore进行安全密钥存储
- **会话管理**: 实现会话状态管理
- **加密/解密**: 处理消息加密和解密

### 2. 后端实现
- **密钥分发**: 提供公钥和预密钥
- **消息路由**: 在用户之间路由加密消息
- **存储**: 存储加密消息和元数据
- **用户在线状态**: 跟踪用户在线状态

### 3. 消息格式
```json
{
  "id": "唯一消息ID",
  "sender": "用户ID",
  "recipient": "用户ID",
  "timestamp": "2023-01-01T00:00:00Z",
  "type": "text",
  "content": "加密消息内容",
  "sessionId": "会话ID",
  "messageKeyId": "消息密钥ID",
  "hmac": "消息认证码"
}
```

### 4. 错误处理
- **会话重置**: 处理会话失效和重新建立
- **密钥不匹配**: 检测并解决密钥验证问题
- **消息解密失败**: 优雅处理解密错误

### 5. 离线支持
- **本地存储**: 本地存储加密消息
- **同步**: 连接恢复时同步消息
- **队列**: 离线时队列消息以进行传递
