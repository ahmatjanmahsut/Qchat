# 数据存储和同步策略

## 概述

本文档概述了聊天应用的数据存储和同步策略，确保消息和用户数据在设备间的高效存储、检索和同步。

## 本地存储

### 1. Room数据库
- **数据库结构**: 基于SQLite的Room数据库
- **消息存储**: 在本地存储加密消息
- **会话元数据**: 存储会话信息
- **用户资料**: 存储用户资料的本地副本
- **加密密钥**: 存储加密的加密密钥

### 2. 数据模型
```kotlin
// 消息模型
@Entity(tableName = "messages")
data class Message(
    @PrimaryKey val id: String,
    val conversationId: String,
    val senderId: String,
    val content: String, // 加密内容
    val type: String, // text, image, video, etc.
    val timestamp: Instant,
    val status: MessageStatus, // sending, sent, delivered, read
    val isOutgoing: Boolean
)

// 会话模型
@Entity(tableName = "conversations")
data class Conversation(
    @PrimaryKey val id: String,
    val type: ConversationType, // private, group
    val name: String,
    val avatar: String?,
    val lastMessageId: String?,
    val lastMessageTimestamp: Instant?,
    val unreadCount: Int
)

// 用户模型
@Entity(tableName = "users")
data class LocalUser(
    @PrimaryKey val id: String,
    val phoneNumber: String,
    val username: String?,
    val displayName: String,
    val profilePicture: String?,
    val publicKey: String,
    val lastSeen: Instant?
)
```

### 3. 本地存储优化
- **索引字段**: 为频繁查询的字段创建索引
- **批量操作**: 对批量插入使用批量操作
- **事务管理**: 对原子操作使用事务
- **数据修剪**: 为旧消息实现自动数据修剪

## 服务器存储

### 1. 数据库模式
- **PostgreSQL数据库**: 用于结构化数据的关系型数据库
- **消息表**: 存储加密消息
- **用户表**: 存储用户信息
- **会话表**: 存储会话元数据
- **群组表**: 存储群组信息
- **成员表**: 存储群组成员关系

### 2. 数据存储考虑
- **消息存储**: 仅存储加密的消息内容
- **元数据存储**: 存储最小化的元数据
- **数据保留**: 定义数据保留策略
- **备份策略**: 实现定期数据库备份

### 3. 服务器存储优化
- **索引**: 正确索引数据库表
- **分区**: 按时间分区大型表
- **缓存**: 使用Redis缓存频繁访问的数据
- **查询优化**: 优化数据库查询

## 同步策略

### 1. 实时同步
- **WebSocket连接**: 使用WebSockets进行实时消息传递
- **消息推送**: 服务器向连接的客户端推送消息
- **在线状态更新**: 实时用户在线状态更新
- **输入指示器**: 实时输入指示器

### 2. 离线同步
- **消息队列**: 离线时队列消息以进行传递
- **重连时同步**: 连接恢复时同步消息
- **冲突检测**: 检测并解决消息冲突
- **增量同步**: 仅同步更改的数据

### 3. 多设备同步
- **设备注册**: 为每个用户注册多个设备
- **跨设备同步**: 在所有用户设备之间同步消息
- **设备特定设置**: 维护设备特定的设置
- **消息状态同步**: 跨设备同步消息状态

## 冲突解决

### 1. 冲突检测
- **消息ID**: 使用唯一消息ID
- **时间戳**: 使用服务器时间戳进行排序
- **版本向量**: 跟踪消息版本
- **基于哈希的去重**: 基于内容哈希去重消息

### 2. 冲突解决策略
- **最后写入者获胜**: 使用服务器时间戳进行冲突解决
- **用户偏好**: 允许用户解决冲突
- **自动合并**: 尽可能合并冲突消息
- **冲突日志**: 记录冲突以进行调试
