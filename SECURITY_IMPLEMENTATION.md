# 安全措施和加密实现

## 概述

本文档概述了聊天应用的安全措施和加密实现，确保所有通信都是安全的，防止未授权访问。

## 加密算法

### 1. 端到端加密
- **算法**: AES-256 in Galois/Counter Mode (GCM)
- **密钥大小**: 256位
- **用途**: 加密所有消息内容
- **实现**: 使用libsignal-protocol-java

### 2. 密钥交换
- **算法**: 扩展三重Diffie-Hellman (X3DH)
- **密钥大小**: 255位 (Curve25519)
- **用途**: 在用户之间安全交换密钥
- **实现**: 使用Signal协议的X3DH实现

### 3. 数字签名
- **算法**: Ed25519
- **密钥大小**: 256位
- **用途**: 验证消息真实性和完整性
- **实现**: 使用libsignal-protocol-java

### 4. 哈希函数
- **算法**: SHA-256
- **用途**: 生成消息摘要和HMAC
- **实现**: 使用Java内置的SHA-256实现

## 密钥管理

### 1. 密钥生成
- **非对称密钥**: 使用Curve25519为X3DH生成
- **对称密钥**: 使用安全随机数生成器生成
- **密钥强度**: 所有密钥使用256位强度
- **生成方法**: 使用Android的KeyGenerator和安全随机源

### 2. 密钥存储
- **私钥**: 存储在Android KeyStore中
- **对称密钥**: 加密并存储在EncryptedSharedPreferences中
- **群组密钥**: 使用用户的私钥加密并存储
- **密钥备份**: 使用用户提供的密码短语进行可选的加密备份

### 3. 密钥交换
- **预密钥包**: 生成并上传到服务器
- **会话建立**: 使用X3DH密钥协商
- **完美前向保密性**: 通过Double Ratchet算法实现
- **密钥验证**: 用户可以通过QR码或指纹验证密钥

### 4. 密钥轮换
- **定期轮换**: 定期轮换密钥
- **基于事件的轮换**: 当安全事件发生时轮换密钥
- **群组密钥轮换**: 当成员加入/离开群组时
- **密钥撤销**: 旧密钥被撤销并替换
